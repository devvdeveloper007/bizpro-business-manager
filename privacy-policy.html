Privacy Policy

Last Updated: April 2026 Version 3.0
Welcome to BizPro — your trusted multi-account business and home manager. This Privacy Policy explains how we collect, use, and protect your information when you use our web-based Progressive Web App (PWA) and any future mobile apps (Android). By using BizPro, you agree to the practices described here.
1
What Information We Collect
  • Admin account: User ID (chosen by you), password, recovery password, admin name — stored in Google Firestore
  • Staff accounts: Names, User IDs, passwords, link IDs (for device-independent login) — stored in Firestore under your Admin namespace
  • Business data: Income entries, expense entries, private ledger folders, to-dos, reminders, notes, inventory items, deleted archive — all stored in Firestore and linked to your unique Firebase UID
  • Authentication: Firebase Auth uses hidden email addresses (e.g., bizproadm_RANDOM@bizproapp.web.app) mapped to your User ID — you never see or use this email directly
  • Device info: NEW If you enable Passkey / Biometric login, we store only a Credential ID (a random string) to identify your device — we never receive your actual fingerprint or Face ID data
  • Usage analytics: We currently do NOT collect any analytics, cookies, or tracking data — no Google Analytics, no third-party scripts
2
How We Use Your Information
  • Authentication: Verify your identity via Firebase Auth (Admin) or custom User ID system (Staff)
  • Data sync: Store and sync all your business entries, staff data, and settings in real time across devices using Google Firestore
  • Multi-user isolation: Ensure Admin and Staff data are properly isolated — Staff cannot see each other's entries or the Admin's private ledger
  • Password recovery: Use your recovery password to reset your main Firebase Auth password if forgotten
  • Passkey login: NEW Store Credential IDs to allow biometric login on supported devices
  • Offline access: Cache static app files (HTML, icons) via Service Worker for offline functionality
3
How We Store Your Information
  • Google Firestore: All your app data (entries, staff, settings, folders, inventory) is stored in Firestore — a NoSQL database hosted on Google Cloud Platform (GCP)
  • Firebase Authentication: Admin passwords are encrypted and managed by Google Firebase Auth — we do not see or store plaintext Admin passwords
  • Firebase App Check: We use Firebase App Check to verify that requests to our backend come only from our genuine app — preventing unauthorized access and bot abuse
  • Password Hashing: Staff passwords are hashed using SHA-256 before storage — we never store plaintext passwords for any user type
  • Staff passwords: Stored as plain text in Firestore accessible only to the Admin within their private namespace. We recommend strong, unique passwords
  • Offline data: Static app files (HTML, icons) are cached by the Service Worker on your device for offline access — no personal data is cached offline
  • Timestamps: NEW All entries now store a server-side Firebase timestamp and a client-side ISO timestamp for accurate time tracking
4
Data Sharing

We do NOT share your personal data with any third parties except:

  • Google Firebase / Firestore: Our backend database and authentication provider — governed by Google's Privacy Policy
  • Google Cloud Platform: Infrastructure provider for Firebase Hosting and Firestore
  • When required by law: If legally compelled by a court order or regulatory requirement in your jurisdiction
5
Multi-User Privacy & Staff Isolation
  • Admin data: Fully private — visible only to the Admin who created it
  • Admin Private Ledger: Completely hidden from all staff — only the Admin can access their private folders
  • Staff entries: Each staff member can only see their own entries — not other staff members' data, names, or balances
  • Staff list hidden: Staff login uses User ID only — no staff member can see the list of other staff members
  • Cross-account isolation: Each registered Admin has a completely isolated workspace — no data is shared between different Admin accounts
  • Password visibility: Admin can view and manage staff passwords within their own workspace — staff cannot see each other's passwords
  • Entry timestamps: NEW All entries (Income, Expense, To-Do, Reminders, Notes) are timestamped — visible to both Admin and relevant Staff
6
Authentication & Passkey / Biometric Login NEW
  • Admin login: Uses a User ID which is mapped to a hidden Firebase email (e.g. bizproadm_abc123@bizproapp.web.app) — the email is never shown to the user. Passwords are encrypted and managed by Google Firebase Auth
  • Recovery password: A backup passphrase stored in Firestore — used to reset the main password. Minimum 6 characters required
  • Staff login: Uses User ID + password system set by Admin — credentials stored in Firestore under Admin's private namespace
  • Password minimum: NEW All passwords (Admin + Staff) require a minimum of 6 characters for better security
  • Passkey / Biometric: Uses the W3C WebAuthn standard — your fingerprint or Face ID stays on your device only. We store only a Credential ID (a random string) to identify your device
  • Staff password change: Staff can change their own password from the login screen — Admin is notified immediately via real-time sync
7
Device Permissions Used
  • Internet / Network access: Required to sync data with Firebase in real time and load the app from Firebase Hosting
  • Biometric hardware (optional): NEW Used only if you enable Passkey login — accessed via the device's secure enclave only. We never receive raw biometric data
  • Browser notifications (optional): Used only if you grant permission — for Reminder due-date alerts only
  • No other permissions are requested or required — no camera, microphone, contacts, location, SMS, or storage access
8
Data Deletion & Account Removal
  • Individual entries: Use the delete button in any module — entries move to Deleted Archive and can be permanently removed from there
  • Edit entries: NEW To-Do tasks, Reminders, and Notes can now be edited directly from the ledger cards without deletion
  • All app data: Admin → Settings → Danger Zone → Reset All Data (permanent, cannot be undone)
  • Remove Passkey: Settings → General → Passkey / Biometric Login → Remove
  • Firebase Auth account deletion: Contact us at devv.developer007@gmail.com — we will delete your account and all associated Firestore data within 30 days
  • Staff member removal: Admin can delete any staff member — their data moves to Deleted Archive and can be permanently deleted
9
Language & Localization Data
  • Language preference: Your selected language (from 21 supported languages including Hindi, Gujarati, Tamil, English, and more) is stored only in your device's localStorage — never sent to any server
  • No location tracking: We do not use your language preference to determine or track your geographic location
  • Supported languages: NEW English, Hindi (हिंदी), Gujarati (ગુજરાતી), Tamil (தமிழ்), Japanese, German, Chinese, French, Spanish, Arabic, Portuguese, Russian, Korean, Italian, Turkish, Dutch, Bengali, Bahasa Indonesia, Vietnamese, Swahili, English (UK)
10
Children's Privacy

BizPro is designed for adults managing business and household finances. This app is not directed to children under the age of 13. We do not knowingly collect personal information from children under 13. If you believe a child has provided us personal data, please contact us immediately at devv.developer007@gmail.com and we will promptly delete it.

11

Continued use of BizPro after changes constitutes acceptance of the updated policy. For significant changes, we will notify users via the app.
